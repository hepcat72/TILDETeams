# TILDETeams
Templates for Iterative Lifecycle Development in Engineering Teams using GitHub

## Mission Statement

GitHub is a platform for social coding, designed to support contributions from unsollicited external contributors that doesn't entirely lend itself to controlled team development, where you have control over every aspect of a project: planning, requirements gathering, design, implementation, testing, and maintenance.  Repository owners have merge-control and can try to guide contributors using pull request and issue templates, but it provides little in the way of standardized internal processes, such as change request approval processes, design language options, design review, requirements tracking, review issue templates, etc. that can benefit team development.

This repository will serve to aid internal team processes by providing boilerplate templates, plans, and guidelines inspired by the Iterative Software Engineering Lifecycle Development Model, guided by the tutelage of an industry software engineering expert, streamlined by decades of configuration management experience in the usage of that model in teams of 3 or more members, and adapted for the GitHub plaform.
